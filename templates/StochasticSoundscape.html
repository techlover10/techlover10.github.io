Stochastic Soundscape

Generate Your World

Stochastic Soundscape takes any sound file, analyzes the patterns of some of its more defining features, and randomly generates an output file which reflects some of these patterns.

StochasticSoundscape on Github!

Background

Inspired by the process-driven music of Iannis Xenakis , the algorithmic processes of Anna Lindemann's biological process-driven music, and stochastic process in general, Stochastic Soundscape is a process-driven soundscape generation tool which creates endless, unique soundscapes based on a set of parameters and given source material.

System Requirements

Currently, the software is written for Linux-based operating systems only. With some slight modifications, it can be made to run on Windows; this is functionality which might be added in the future. The following dependencies are also required:

  • Python 3.5
  • pydub
  • scipy
  • numpy
  • wave
  • Librosa
  • ffmpeg

For more info, check out the README on GitHub

Sample Generation

It is rather unlikely that users have a volume of samples that is sufficient for demonstrating what this program can do. To generate samples, sample_clipper.py contains code that generates samples using the same onset or beat detection that is used for analysis. This program, which does not interface with the main process of generating soundscapes, will clip given source material into samples which can then be classified and used for soundscape genration.

Process

Stochastic Soundscape uses two types of source material, structural and samples. Structural data is analyzed to generate the stochastic transition matrix from which the samples will be assembled. The basic idea behind the process is simple: first, Librosa is used to detect onsets (any sort of noticeable bump in the waveform of the sound) in the structural material. The sound is then iterated through from onset to onset, and each sample is classified (see Feature Analysis for more info). Ideally, the onset intervals will divide the soundscape into small sounds, although in practice sound is messy and this is not necessarily true. Each time a snippet is analyzed, the transition between the previous classifier and the current one is logged into a Markov transition matrix . The samples are classified using the same methodology as the structural source material. Finally, a soundscape is assembled randomly, using transition probabilities from the structural material to choose sounds from the samples.

Feature Analysis

Classification

Librosa includes various sound analysis functions, and Stochastic Soundscape uses many of them in its default method of classification. A specific classification method can be chosen by specifying which type to run in the settings file (settings.py).

  • 'rolloff' analyzes sounds based on spectral rolloff, a feature which is based on how much energy is in which frequency ranges for the sound.
  • 'spectral_centroid' analyzes the "center of mass" of a sound, which reflects the "brightness" of the sound.
  • 'zero_crossing' measures the rate at which the sign of the signal changes - that is, the rate at which the signal changes from positive to negative.

Pulse Detection

As mentioned above, Stochastic Soundscape defaults to using onset detection as a method of defining snippet boundaries for analysis. Another option is to use Librosa's "beat detection", which essentially performs onset detection with the addition of attempting to infer a tempo, and ultimately cut on beats. This is useful for experimental music, or experimenting with any sort of rhythmic sound.

Frequency Bands

By default, Stochastic Soundscape analyzes sounds in three frequency bands, to add an additional level of randomness. The frequency boundaries can be adjusted, and this can also be entirely disabled. All of this is also controlled in settings.py.
